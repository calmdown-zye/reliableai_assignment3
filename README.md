# Assignment 3: Neural Network Verification with Marabou

**Course**: Reliable and Trustworthy Artificial Intelligence  
**Topic**: Local robustness verification of a small MNIST classifier using Marabou

---

## Overview

This project trains a small fully-connected network on MNIST, exports it to ONNX,
and verifies its local $\ell_\infty$ robustness using [Marabou](https://github.com/NeuralNetworkVerification/Marabou).

**Verification query**: For a test image of digit 7, prove that all inputs within
$\|\mathbf{x}' - \mathbf{x}\|_\infty \leq 0.01$ are also classified as digit 7.

## Environment Setup

Python 3.8 with conda is recommended.

```bash
conda create -n marabou python=3.8 -y
conda activate marabou
pip install -r requirements.txt
```

**requirements.txt** includes:
- `torch`, `torchvision` (CPU build)
- `onnx`, `onnxruntime`
- `maraboupy`
- `numpy`

> **Note**: The default MNIST mirror (`yann.lecun.com`) returns 404.
> `train_model.py` overrides the mirror to use Google Storage automatically.

---

## How to Run

### Step 1: Train the model and export to ONNX

```bash
python3 scripts/train_model.py
```

This downloads MNIST (if not present), trains the model for 5 epochs,
and saves `models/mnist_fc.onnx`.

### Step 2: Run the Marabou verification

```bash
python3 scripts/verify.py
```

This runs 9 pairwise robustness queries (digit 7 vs. each other class)
with $\varepsilon = 0.01$ and prints SAT/UNSAT results with runtimes.

### Step 3: Run the grading entry point

```bash
python3 test.py
```

## Installation Notes

- Marabou's ONNX parser requires `onnxruntime` in addition to `maraboupy`.
  Without it, `Marabou.read_onnx()` raises a `NameError`.
- TensorFlow parser warnings (`Tensorflow parser is unavailable`) are expected
  and can be safely ignored.
